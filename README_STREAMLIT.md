# 솔루미랩 경리나라 전표 생성 앱 (Streamlit 버전)

매출/매입 전표 날짜별 일괄등록 파일을 자동으로 생성하는 웹 애플리케이션입니다.

## 🚀 설치 방법

### 최초 설치 (1회만)

1. **Python 가상환경 활성화**
   ```bash
   # Mac/Linux
   source bin/activate

   # Windows
   .\Scripts\activate
   ```

2. **필요한 패키지 설치**
   ```bash
   pip install -r requirements.txt
   ```

3. **완료!**

---

## ▶️ 실행 방법

### 방법 1: 스크립트 실행 (추천)

**Mac/Linux:**
```bash
./start.sh
```

**Windows:**
```
start.bat 더블클릭
```

### 방법 2: 직접 실행

```bash
# 가상환경 활성화 후
streamlit run app.py
```

실행하면 자동으로 브라우저가 열립니다.
기본 주소: `http://localhost:8501`

## 📖 사용 방법

### 1. 전표 생성

1. **파일 선택**
   - "발주내역 파일 선택" 클릭
   - `.xlsm` 파일 선택
   - 파일에 `(누적)2025년 발주내역` 시트가 있어야 함

2. **날짜 선택**
   - 처리할 날짜를 캘린더에서 선택

3. **처리 시작**
   - "▶️ 업로드 및 처리" 버튼 클릭
   - 진행 상황이 표시됨

4. **결과 확인 및 다운로드**
   - 매출 데이터 미리보기
   - 매입 데이터 미리보기
   - "📥 매출 파일 다운로드" 클릭
   - "📥 매입 파일 다운로드" 클릭

### 2. 처리 이력 확인

- 좌측 메뉴에서 "처리 이력" 선택
- 이번 세션 처리 내역 확인
- 저장된 파일 목록 확인
- 애플리케이션 로그 확인

### 3. 설정

- 좌측 메뉴에서 "설정" 선택
- 거래처마스터 파일 경로 확인
- 디스크 사용량 확인
- 오래된 파일 정리

## 🛠️ 문제 해결

### 문제 1: "거래처마스터 파일을 찾을 수 없습니다"

**원인:** `Src/거래처마스터.xlsx` 파일이 없음

**해결:**
1. `Src` 폴더에 `거래처마스터.xlsx` 파일이 있는지 확인
2. 파일명이 정확한지 확인 (띄어쓰기, 확장자 확인)
3. 설정 페이지에서 파일 경로 확인

### 문제 2: "Excel 시트를 찾을 수 없습니다"

**원인:** 업로드한 파일에 `(누적)2025년 발주내역` 시트가 없음

**해결:**
1. 올바른 파일을 선택했는지 확인
2. Excel에서 파일을 열어 시트 이름 확인
3. 시트 이름이 정확히 `(누적)2025년 발주내역`인지 확인

### 문제 3: 앱이 시작되지 않음

**원인:** Streamlit이 설치되지 않음

**해결:**
```bash
# 가상환경 활성화
source bin/activate  # Mac/Linux
# 또는
.\Scripts\activate  # Windows

# Streamlit 설치
pip install streamlit

# 앱 실행
streamlit run app.py
```

### 문제 4: 브라우저가 자동으로 열리지 않음

**해결:**
수동으로 브라우저를 열고 아래 주소로 접속
```
http://localhost:8501
```

## 📞 원격 지원

### 개발자에게 문제 보고하는 방법

1. **스크린샷 캡처**
   - 에러 메시지가 표시된 화면 캡처
   - "🔍 에러 상세 정보" 부분도 펼쳐서 캡처

2. **로그 다운로드**
   - 좌측 메뉴 → "처리 이력" 선택
   - 아래로 스크롤해서 "애플리케이션 로그" 섹션으로 이동
   - "📥 전체 로그 다운로드" 클릭

3. **개발자에게 전송**
   - 스크린샷 + 로그 파일 전송
   - 어떤 작업을 했는지 간단히 설명
   - 언제 에러가 발생했는지 시간 기록

### 개발자가 원격으로 확인하는 방법

1. **로그 파일 확인**
   - 전송받은 로그 파일에서 에러 원인 파악

2. **SSH 접속** (NAS 배포 시)
   ```bash
   ssh admin@nas-ip
   cd /volume1/docker/sollume-finance

   # 로그 확인
   tail -f logs/app.log

   # 컨테이너 상태 확인
   docker-compose ps

   # 컨테이너 로그 확인
   docker-compose logs -f
   ```

3. **문제 해결 후 재시작**
   ```bash
   # 코드 수정 후
   docker-compose restart
   ```

│   └── app.log               # 애플리케이션 로그
├── uploads/                  # 임시 업로드 폴더
├── processed/                # 처리된 파일 저장
├── requirements.txt          # Python 패키지 목록
├── start.sh                  # Mac/Linux 실행 스크립트
├── start.bat                 # Windows 실행 스크립트
└── README_STREAMLIT.md       # 이 파일

## 🔐 보안 주의사항

1. **외부 접속 제한**
   - 기본적으로 `localhost`에서만 접속 가능
   - 외부 접속이 필요한 경우 방화벽 설정 필요

3. **마스터 파일 관리**
   - `거래처마스터.xlsx` 파일 변경 시 주의
   - 변경 전 백업 권장
